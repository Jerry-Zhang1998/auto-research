# generate-report

Generate all HTML output files for a paper. Reproduction code lives in `outputs/{name}/reproduction/` and was generated by Stage 3 (`/reproduce-code`).

Output: `outputs/{name}/html/summary.html`, `outputs/{name}/html/train.html` (if logs exist), `outputs/{name}/html/evaluate.html` (if test_results.json exists)

## Arguments

$ARGUMENTS

First argument: paper name slug (e.g. `attention-is-all-you-need`). Must have `analyses/{name}/innovations.md`.

## Steps

**Step 1 — Load inputs.**

Read these reference files before generating anything:
1. `analyses/{name}/innovations.md` — primary source. If missing: tell user to run `/analyze-innovations {name}` first.
2. `analyses/{name}/raw.md` — supplemental detail.
3. `prompts/html_report_system.md` — HTML design rules (dark theme, self-contained, no CDN).
4. `outputs/_template/html/summary.html` — **reference template**: follow its CSS variables, class names, and section structure exactly.
5. `outputs/_template/html/evaluate.html` — shows the metric card + ROC/PR chart layout.
6. `outputs/_template/html/train.html` — shows the loss curve + metric chart layout.

After reading `innovations.md`, extract the `sections_detected` dict from its YAML frontmatter and the paper type/domain from the `## Paper Profile` section. Store as `{MANIFEST}`. This controls which HTML sections to render:

| Key | HTML section to include |
|-----|------------------------|
| `theoretical_analysis: true` | Include `<section id="theory">` block |
| `model_architecture: true` | Include `<section id="architecture">` block |
| `loss_design: true` | Include `<section id="loss">` block |
| `training_strategy: true` | Include `<section id="training">` block |
| `efficiency_analysis: true` | Include `<section id="efficiency">` block |
| `ablation_study: true` | Include ablation card inside `<section id="results">` |
| `implementation_notes: true` | Include `<section id="implementation">` block |

Sections with `false` are **completely omitted** from the HTML — no empty blocks, no "N/A" text.

**Step 2 — Create output directories.**

Run: `mkdir -p outputs/{name}/html outputs/{name}/reproduction`

**Step 3 — Extract architecture figure.**

Before generating the HTML, check for an extracted figure:

```bash
python3 scripts/extract_figures.py papers/{name}.pdf analyses/{name}/
```

Parse the JSON output and get `arch_figure`. If `arch_figure.has_b64` is `true`,
the base64 PNG lives in the sidecar file at `arch_figure.b64_file` (the blob is
NOT in stdout — that would flood context). Record `ARCH_CAPTION` from
`arch_figure.caption`.

**Do not read the b64 sidecar or the PNG into your own context.** The HTML
generator script (Step 4) reads the bytes directly from disk and base64-encodes
them itself — exactly as `/tmp/gen_*_summary.py` does with
`base64.b64encode(open(png,'rb').read())`. You only need the path and caption.

If `extract_figures.py` fails (PyMuPDF not installed) or `has_b64` is false, the
architecture section falls back to the ASCII diagram only (no `<img>`).

**Step 4 — Generate `outputs/{name}/html/summary.html`.**

Write a complete, self-contained HTML file. All CSS must be inlined in `<style>` — no external URLs, no CDN links. The file must render correctly when opened directly in a browser without internet access.

**Dynamic section rendering rules** (based on `{MANIFEST}` from Step 1):
- Render the `<div class="profile-bar">` with paper type, domain, and section chips (`chip-on` / `chip-off`)
- Build the TOC `<ol>` with **only** the sections that are present
- Include the `<section id="theory">` block **only if** `theoretical_analysis = true`
- Include the `<section id="architecture">` block **only if** `model_architecture = true`
- Include the `<section id="loss">` block **only if** `loss_design = true`
- Include the `<section id="training">` block **only if** `training_strategy = true`
- Include the `<section id="efficiency">` block **only if** `efficiency_analysis = true`
- Inside `<section id="results">`: include the ablation card **only if** `ablation_study = true`
- Include `<section id="implementation">` **only if** `implementation_notes = true`

**HTML structure** — use `outputs/_template/html/summary.html` (already read in Step 1) as the exact reference for CSS variables, class names, section layouts, and new component styles (`.profile-bar`, `.chip-on/.chip-off`, `.theory-claim`, `.derivation`, `.efficiency-grid`, etc.). Reproduce the full CSS from that file inline in `<style>`. Do not use external links.

Section order in the generated file:
1. `<div class="paper-header">` — title, authors, year, arXiv badge, TL;DR
2. `<div class="profile-bar">` — paper type, domain, section presence chips
3. `<nav class="toc">` — only list the sections actually rendered below
4. `<section id="problem">` — always present
5. `<section id="theory">` — **only if** `theoretical_analysis = true`; use `.theory-claim`, `.derivation`, `.assumption-list` blocks
6. `<section id="contributions">` — always present
7. `<section id="architecture">` — **only if** `model_architecture = true`; embed `ARCH_B64` if available
8. `<section id="loss">` — **only if** `loss_design = true`
9. `<section id="training">` — **only if** `training_strategy = true`
10. `<section id="efficiency">` — **only if** `efficiency_analysis = true`; use `.efficiency-grid` with four `.efficiency-card` blocks
11. `<section id="results">` — always present; include ablation card only if `ablation_study = true`
12. `<section id="implementation">` — **only if** `implementation_notes = true`
13. `<footer>` — "Generated by auto-research · {DATE}"

Populate every field with actual content from `innovations.md`. Do not leave any `{PLACEHOLDER}` text in the final output. Write the complete file to `outputs/{name}/html/summary.html`.

```html
<!-- structural skeleton — see outputs/_template/html/summary.html for full CSS -->
<!DOCTYPE html><html lang="en"><head><meta charset="UTF-8">
<title>{title} — Research Summary</title>
<style>/* ← reproduce full CSS from outputs/_template/html/summary.html here */</style>
</head><body>
<div class="paper-header">...</div>
<div class="profile-bar">...</div>
<nav class="toc"><ol><!-- only present sections --></ol></nav>
<section id="problem">...</section>
<!-- conditional sections in order: theory, contributions, architecture, loss, training, efficiency, results, implementation -->
<footer>Generated by auto-research · {DATE}</footer>
</body></html>
```

**Step 5 — Verify `outputs/{name}/reproduction/` code.**

The reproduction code was generated in Stage 3 (`/reproduce-code`) and lives in `outputs/{name}/reproduction/`. If Stage 3 was skipped or the directory is missing, generate the 7 files (config.py, model.py, loss.py, dataset.py, train.py, test.py, README.md) following the same rules as `reproduce-code.md`. The `model.py` in this directory may have local imports (e.g. `from config import ModelConfig`) — that is expected and correct since all files share the same directory.

**Step 6 — Generate training and evaluation visualisations.**

Check whether training logs exist for this paper:

```bash
find logs/{name} -name "metrics.jsonl" | sort | tail -1
find logs/{name} -name "test_results.json" | sort | tail -1
```

- **If `metrics.jsonl` found**: run `python3 scripts/generate_viz.py --log-dir <that_dir> --output-dir outputs/{name}/html` to generate `train.html` and `evaluate.html`.
- **If `test_results.json` not found**: note that `evaluate.html` will be skipped; the user should run `python test.py` in `outputs/{name}/reproduction/` first, then re-run `/generate-report {name}`.
- **If no logs at all**: skip this step and note it in the confirmation output.

**Step 7 — Confirm.**

Run `find outputs/{name} -type f | sort` to list output files.

Print:
```
✓ Report generated
  outputs/{name}/
  ├── html/
  │   ├── summary.html    innovation analysis (self-contained dark-theme HTML)
  │   ├── train.html      training curves (loss, AUC, LR)         [if logs exist]
  │   └── evaluate.html   ROC/PR curves, confusion matrix, metrics [if test_results.json exists]
  └── reproduction/        (generated by Stage 3 — /reproduce-code)
      ├── config.py
      ├── model.py
      ├── loss.py
      ├── dataset.py
      ├── train.py
      ├── test.py
      └── README.md

  Open in browser:
    open outputs/{name}/html/summary.html
    open outputs/{name}/html/train.html
    open outputs/{name}/html/evaluate.html
```
